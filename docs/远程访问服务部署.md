# 远程访问服务部署

远程访问服务用于手机扫码申请、白名单审批、移动监看、受限 ATEM 操作和直播间内部更新文件分发。服务应部署在可信局域网内，并使用独立管理密码。

## 服务边界

- 容器仅监听服务器的 `192.168.110.111:8088`。
- 容器根文件系统只读，移除全部 Linux capabilities，并启用 `no-new-privileges`。
- 仅挂载 `data` 与 `updates` 两个专用目录，不挂载宿主机其他路径。
- 手机端只允许选择 PVW 和执行二次确认后的 AUTO，不提供 Hard Cut。
- 二维码只用于提交申请；未经后台审批，浏览器不能读取状态或发送命令。

## 配置

在 `remote-server` 目录创建 `.env`：

```dotenv
PORT=8088
PUBLIC_BASE_URL=http://192.168.110.111:8088
ADMIN_PASSWORD=请替换为独立强密码
DATA_DIR=/data
UPDATE_DIR=/updates
```

启动服务：

```bash
docker compose up -d --build
```

管理后台地址为 `http://192.168.110.111:8088/admin`，健康检查地址为 `http://192.168.110.111:8088/health`。

## 更新文件

管理员登录后可在“内部更新文件”区域上传或删除安装包、版本清单和 blockmap。桌面端选择“直播间内部更新服务器”，或使用自动更新源时，会从 `/updates/` 读取对应平台的更新文件。

## 运维建议

- 管理密码不要与 Linux、NAS、OBS 或其他服务共用。
- 仅允许可信办公网或直播网访问 8088 端口；如需跨公网使用，应增加 HTTPS 反向代理、域名和访问控制。
- 定期备份 `remote-server/data`，其中保存设备、申请和白名单记录。
- 更新服务后执行 `docker compose ps` 和 `curl http://127.0.0.1:8088/health` 确认容器健康。
