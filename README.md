# OBS 音频检测助手

English name: OBS Audio Monitor Assistant

一个面向直播现场的 Windows OBS 伴随程序，重点用于电商直播、带货直播、活动直播等场景。它会监听指定 OBS 音频源，当主播麦克风、无线领夹麦、声卡输入或主混音连续静音超过设定时间时，先显示预警浮窗，再弹出醒目的置顶提醒，帮助主播或导播及时发现麦克风没电、被静音、掉线、路由异常等问题。

## 适用场景

- 电商直播中主播麦克风没电、静音键误触、接收器掉线，直播间长时间没有人声。
- 导播间需要持续盯住 OBS 主麦、嘉宾麦、声卡输入或节目主混音。
- 长时间直播时，希望在不打断操作的情况下获得静音预警和正式报警。
- 直播或录制开始后才检测，OBS 空闲时不误报。

## 功能

- 通过 OBS 28+ 内置 WebSocket 连接 OBS。
- 选择一个 OBS 输入源作为检测对象，例如主播麦克风、无线领夹麦、声卡输入或直播主混音。
- 默认仅在直播或录制中检测。
- 默认静音阈值 `-55 dBFS`，默认静音时长 `120 秒`，都可调整。
- 默认静音达到 75% 时显示屏幕中下方预警浮窗，120 秒时再弹正式报警，适合电商直播中“短暂停顿不打扰、长时间无声才提醒”的节奏。
- 中央非全屏红色置顶弹窗，避免全屏惊吓。
- 弹窗支持 `确定`、`单次忽略`。确定会关闭弹窗并重新累计静音时长；单次忽略会关闭弹窗，并在后台 5 分钟后自动恢复检测。
- 后台设置页和托盘菜单都支持暂停/恢复检测。
- 主窗口点击关闭按钮会隐藏到后台，彻底退出需要从系统托盘菜单选择 `退出`。
- 检测到多屏时支持主屏、指定屏幕、所有屏幕弹窗策略；单屏时默认在当前屏幕显示，不额外展示屏幕设置。
- 支持记住拖动后的报警弹窗位置。
- 支持 OBS 连接测试和最近 20 条报警历史记录。
- 设置页提供 `测试报警弹窗`，不用连接 OBS 也能确认弹窗位置和样式。
- 首次启动会自动显示操作说明，之后也可以在设置页点击 `操作说明` 再次打开。

## 开发运行

```bash
npm install
npm run dev
```

构建后本地启动：

```bash
npm start
```

OBS 中需要开启 WebSocket：

1. 打开 OBS。
2. 进入 `工具` -> `WebSocket 服务器设置`。
3. 启用 WebSocket 服务器。
4. 默认端口通常是 `4455`。
5. 如果设置了密码，在软件设置页填写同一个密码。

## 构建

```bash
npm run build
```

Windows 安装包：

```bash
npm run dist:win
```

产物会输出到 `release/`。

如果只需要拿到一个可双击测试的 Windows 便携版：

```bash
npm run package:win-portable
```

生成后把 `release/OBS-Audio-Monitor-Assistant-版本号-win-x64-portable.zip` 拷到 Windows 解压，双击里面的 `OBS 音频检测助手.exe`。

## 验证

```bash
npm test

```
