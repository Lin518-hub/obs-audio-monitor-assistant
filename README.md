# OBS 音频检测助手

**OBS Audio Monitor Assistant** 是一款为直播现场准备的 OBS 音频守护工具。它会持续监听指定的 OBS 音频源，在主播麦克风、无线领夹麦、声卡输入或直播主混音长时间没有声音时，先给出轻量预警，再弹出醒目的置顶提醒，帮助主播和导播及时发现问题。

它特别适合电商直播、带货直播、活动直播、访谈直播等不能长时间无声的场景。

## 为什么需要它

直播事故里最难发现的一类问题，是画面正常但麦克风已经没声：无线麦没电、静音键误触、接收器掉线、声卡路由异常、OBS 音源选错，甚至主播自己完全不知道观众已经听不到声音。

OBS 音频检测助手做的事很简单：把这类问题从“观众提醒你”提前到“软件提醒你”。

## 核心能力

- **直播中自动监听**  
  默认只在 OBS 正在直播或录制时检测，开播前调试不会误报。

- **只关注可能有声音的来源**  
  音源列表会过滤图片、文字、显示器采集等通常无声来源，只保留更可能产生声音的麦克风、声卡、媒体源、浏览器源或采集设备。

- **静音预警 + 正式报警**  
  默认连续静音 90 秒显示小浮窗预警，120 秒弹出正式报警。时间和阈值都可以调整。

- **醒目但不吓人的报警窗**  
  报警窗不是全屏遮挡，而是现代化置顶弹窗。导播还能看到后面的 OBS 或直播后台。

- **小浮窗常驻观察**  
  可以把小浮窗放在屏幕角落，随时查看静音时间、当前状态和电平。小浮窗支持独立深色/浅色切换。

- **后台托盘运行**  
  关闭主窗口后软件仍在后台检测，需要退出时从系统托盘菜单选择退出。

- **报警历史**  
  本地保留最近 20 条报警记录，方便复盘是哪一路音源、持续多久、当时做了什么处理。

## 适用场景

- 电商直播中，主播麦克风没电或接收器掉线。
- 主播误触静音键，直播间长时间没有人声。
- 导播需要同时盯 OBS、直播后台和音频状态。
- 声卡、采集卡或 OBS 音频路由偶发异常。
- 长时间直播中，希望把音频事故变成可见、可处理的提醒。

## 快速开始

1. 打开 OBS。
2. 进入 `工具` -> `WebSocket 服务器设置`。
3. 启用 WebSocket 服务器，默认端口通常是 `4455`。
4. 打开 OBS 音频检测助手。
5. 在软件里填写 OBS 主机、端口和密码。
6. 选择要监听的音源，例如主播麦克风、无线领夹麦、声卡输入或直播主混音。
7. 开始直播或录制后，软件会自动进入检测。

首次打开软件时，会出现 4 步引导：当前状态、OBS 连接、目标音源、快捷操作。也可以随时在“诊断与测试”里重新打开操作说明。

## 日常使用

- **当前状态**：顶部会直接显示 OBS 未连接、等待直播、检测中、静音预警、正在报警等状态。
- **暂停检测**：临时调试或中场休息时可以暂停，恢复后继续检测。
- **小浮窗**：适合放在屏幕角落，直播中快速观察静音时间和电平。
- **诊断与测试**：连接测试、刷新音源、测试报警弹窗都收在这里，不干扰日常操作。
- **确定**：关闭报警并重新开始累计静音时间。
- **单次忽略**：关闭本次报警，后台延后 5 分钟后自动恢复检测。

## 安全与隐私

OBS 音频检测助手只连接本机或你填写的 OBS WebSocket 地址，用来读取 OBS 状态、输入源列表和音频电平。报警历史保存在本地，不上传、不联网同步。

## 系统要求

- Windows 10 或 Windows 11
- OBS Studio 28 或更高版本
- OBS WebSocket 已启用
