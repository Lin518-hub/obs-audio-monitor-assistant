# OBS 音频检测助手

**OBS Audio Monitor Assistant** 是一款面向直播现场的 OBS 音频状态检测工具，用于持续监听指定音频源，并在音频长时间无有效电平时提醒主播、导播或现场运营人员及时处理。

该工具重点服务于电商直播、品牌直播、活动直播、访谈直播及其他对音频稳定性要求较高的直播场景。

## 项目定位

直播过程中，画面正常但麦克风无声是一类较难被现场人员第一时间发现的问题。常见原因包括无线麦克风电量耗尽、静音键误触、接收器断连、声卡路由异常、OBS 音频源选择错误或音源未处于活动状态。

OBS 音频检测助手通过连接 OBS WebSocket，读取 OBS 状态、输入源列表和目标音源电平。当目标音源在直播或录制期间连续低于设定阈值时，系统会先给出预警，再触发报警弹窗，帮助现场人员在观众反馈前发现并处理音频异常。

## 主要功能

- **OBS WebSocket 连接**
  - 支持配置主机、端口和 WebSocket 密码。
  - 支持连接测试，明确提示连接成功、端口错误、密码错误或输入源读取失败等情况。

- **多音源检测**
  - 从 OBS 输入源中筛选可能产生声音的来源。
  - 支持监听麦克风、声卡输入、系统音频、媒体源、浏览器源和采集设备等音频相关来源。
  - 支持同时选择多路音源，任一路被选中音源连续静音超时都会报警，适合主播、嘉宾、场外麦或主混音同时守护。

- **直播状态识别**
  - 默认仅在 OBS 正在直播或录制时进入检测。
  - 未开播、未录制、未选择音源、目标音源不存在或暂时没有电平数据时，不触发静音报警。

- **静音计时与报警**
  - 默认连续静音 120 秒后报警。
  - 静音达到设定时长的 75% 时显示预警浮窗。
  - 3 秒以内的短暂停顿仍视为正常讲话，超过 3 秒后进入静音计时。

- **小浮窗**
  - 支持将检测状态以小窗口形式置顶显示。
  - 小浮窗独立支持深色和浅色显示。
  - 正常状态显示“检测中 / 正在讲话 / 音频正常”，静音超过 3 秒后显示静音计时和剩余报警时间。
  - 提供音频提醒、音频与机位、多功能监看三种模式；多功能模式会根据启用模块数量自适应排版。
  - 支持按比例拖动缩放，并可选择显示 ATEM 当前机位、机位停留时间和 OBS 性能摘要。

- **报警弹窗**
  - 弹窗提供“确定”和“单次忽略”两种处理方式。
  - 点击“确定”后关闭弹窗，并重新开始静音计时。
  - 点击“单次忽略”后关闭弹窗，并在后台延后 5 分钟恢复检测。
  - 支持经典报警弹窗，或在桌面内容上方显示透明渐变红边并保留中央报警弹窗。
  - 可开启声音提示，通过系统默认输出设备播放提示音。

- **后台运行**
  - 关闭主窗口后程序继续在后台运行。
  - 需要完全退出时，可通过系统托盘或菜单栏图标退出。

- **报警历史**
  - 本地保存最近 20 条报警记录。
  - 记录报警时间、音源名称、静音持续时间、用户操作和当时状态。

- **监控面板 Beta**
  - 显示 OBS `GetStats` 性能数据，包括 CPU、内存、FPS 和丢帧统计。
  - 显示所有音频输入设备及当前电平状态。
  - 显示最近音量历史和本次运行中的静音事件日志，便于直播后回溯。

- **ATEM 导播台 Beta**
  - 支持通过局域网连接 Blackmagic ATEM 导播台。
  - 可显示当前 PGM / PVW 机位和输入通道名称。
  - 支持扫描同网段疑似 ATEM 设备，并选择设备 IP。
  - 支持当前 PGM 机位停留计时，默认 10 分钟后在小浮窗中提示。
  - 本地记录每次 PGM 切换的来源、目标机位、切换时间和上一机位停留时长。
  - 按直播场次生成机位切换时间轴与使用占比，本地自动保留最近 10 场。
  - 支持为常用信号设置自定义名称、颜色和分组。
  - 支持 Preview、AUTO Transition 与带确认保护的 Hard Cut。
  - 该功能仍处于 Beta 阶段，默认不启用硬件连接。

- **软件更新**
  - 可优先从直播间内部更新服务器获取安装包，离开内网后自动回退到已配置的镜像或 GitHub Releases。
  - Windows 版本支持下载后重启安装。
  - macOS 当前未进行 Apple Developer 签名，下载后会打开安装包位置，由用户手动替换 App。

- **手机远程访问**
  - 电脑端生成专属二维码，手机或平板无需安装 App，使用浏览器即可打开移动监看页面。
  - 移动端可查看音频状态、静音计时、OBS 性能摘要、ATEM 当前机位与机位停留时间。
  - ATEM 远程控制仅开放选择 PVW 和二次确认后的 AUTO，不开放远程 Hard Cut。
  - 每台电脑拥有稳定 UUID；首次扫码需填写直播间名称并等待管理员审批。
  - 管理后台可统一审批、拒绝或撤销移动浏览器的访问权限，并管理内部更新文件。

## 首次配置

首次打开软件时，界面会显示新手引导。引导会按照实际界面位置说明关键区域，包括当前状态、OBS WebSocket 连接、端口与密码配置、连接测试、目标音源选择、报警规则、小浮窗和常用操作。软件更新到新版本后会再次显示引导，已有配置和本地数据不会被清空，可按需逐步查看或直接下一步完成。

如需再次查看说明，可在软件中的“诊断测试”区域打开“查看说明书”。

## OBS 设置说明

1. 打开 OBS Studio。
2. 在 OBS 顶部菜单进入 `工具` -> `WebSocket 服务器设置`。
3. 启用 WebSocket 服务器。
4. 确认端口，OBS 默认端口通常为 `4455`。
5. 如 OBS 设置了 WebSocket 密码，请在本软件中填写相同密码。
   - 开启“安全保存 OBS 密码”时，软件会使用 Windows 凭据保护或 macOS 钥匙串加密保存。
   - macOS 询问“登录”钥匙串密码属于系统授权流程；不希望保存时可关闭该选项，密码只在本次运行中使用。
6. 回到本软件，点击连接测试，确认可读取 OBS 状态和输入源列表。
7. 在“目标音源”中选择需要守护的主播麦克风、无线领夹麦、声卡输入或直播主混音。需要同时监控多路麦克风时，可在设置中勾选多路音源。

OBS 28 及以上版本通常已内置 WebSocket 功能；旧版本可能需要单独安装 OBS WebSocket 插件。

## 日常使用流程

1. 启动 OBS 并确认 WebSocket 已启用。
2. 打开 OBS 音频检测助手。
3. 确认顶部状态显示 OBS 已连接。
4. 选择需要检测的目标音源。
5. 根据直播间节奏调整静音时长和静音阈值。
6. 开始直播或录制后，软件会自动进入检测。
7. 直播过程中可打开小浮窗，持续观察音频状态。

## 手机远程访问

1. 在设置中进入“手机远程”，确认服务器地址并启用手机扫码访问。
2. 等待状态显示“远程服务已连接”，再使用手机或平板扫描二维码。
3. 首次扫码输入直播间名称和设备名称，提交访问申请。
4. 管理员在内部管理后台审批后，当前浏览器会自动进入移动监看页面。
5. 已批准的浏览器会在本机保存访问凭据；管理员可随时在后台撤销权限。
6. 移动端切台时，先点击信号源选入 PVW，再通过确认面板执行 AUTO。

若公网电脑提示域名不存在或浏览器显示 `DNS_PROBE_FINISHED_NXDOMAIN`，请先执行 `ipconfig /flushdns`，并将网络 DNS 改为 `223.5.5.5` 或 `119.29.29.29` 后重试。该错误发生在域名解析阶段，尚未连接到远程服务器。

远程访问默认关闭。建议仅在可信直播局域网内启用，并为管理后台设置独立强密码。二维码只能用于提交申请，不能绕过管理员审批直接查看或切台。

## ATEM Beta 使用说明

1. 确认 ATEM 导播台和电脑位于同一局域网。
2. 在左侧进入 `ATEM 导播台`，点击 `配置 ATEM`。
3. 在设置中启用 `ATEM 连接`。
4. 如已知导播台 IP，可直接填写；如不确定，可点击 `查找导播台` 扫描同网段设备。
5. 连接成功后，主界面会显示 PGM、PVW 和输入通道名称。
6. 在 ATEM 页面中，数字键 `1-8` 可选择 PVW 机位，`Enter` 执行 AUTO 切换。启用全局切台快捷键后，该操作在其他软件中同样有效，直播前应先完成测试。
7. Hard Cut 默认带确认保护，建议在非直播环境测试快捷键和切换逻辑后再用于直播。

该功能为 Beta 版本，建议先在非直播环境中完成连接和切换测试。

## 状态说明

| 状态 | 含义 |
| --- | --- |
| OBS 未连接 | 未能连接 OBS WebSocket，请检查 OBS 是否打开、WebSocket 是否启用、端口或密码是否正确。 |
| 等待直播/录制 | OBS 已连接，但当前未直播或录制，因此不会触发静音报警。 |
| 检测中 | OBS 已进入直播或录制，目标音源处于检测状态。 |
| 正在讲话 | 目标音源有有效电平，或短暂停顿未超过 3 秒。 |
| 静音计时中 | 目标音源低于阈值超过 3 秒，正在累计静音时长。 |
| 正在报警 | 目标音源连续静音达到设定时长，需要现场人员确认。 |
| 已延后检测 | 用户选择单次忽略后，系统在指定时间后自动恢复检测。 |
| 检测已暂停 | 用户手动暂停检测，不会触发静音报警。 |

## 报警处理

- **确定**：确认本次报警，关闭弹窗，并重新开始静音计时。
- **单次忽略**：关闭弹窗，并在后台延后 5 分钟后恢复检测。
- **暂停检测**：适合中场休息、调试设备或暂时不需要检测的场景。

报警弹窗不会覆盖整个屏幕，方便导播在处理提醒的同时查看 OBS、直播后台或其他工作窗口。

## 适用场景

- 电商直播中主播麦克风没电、接收器断连或声卡输入异常。
- 导播需要同时观察 OBS、直播后台和音频检测状态。
- 长时间直播中，需要将音频异常转化为明确的现场提醒。
- 活动直播、访谈直播或培训直播中，需要持续确认主讲人音频是否正常。

## 数据与隐私

OBS 音频检测助手仅连接用户配置的 OBS WebSocket 与 ATEM 地址。报警历史保存在本地设备，不会上传至远程服务。

启用手机远程访问后，电脑会主动向用户配置的远程服务发送用于移动监看的必要状态摘要，包括当前音频源名称与电平、静音状态、OBS 运行摘要、ATEM PGM/PVW 与机位计时。服务不会接收 OBS WebSocket 密码、报警历史或完整软件配置。移动访问必须经过管理员审批，管理员可随时撤销白名单。

## 系统要求

- Windows 10 或 Windows 11
- macOS Apple Silicon 设备（M 系列芯片，arm64）
- 不支持 Intel x86 Mac
- OBS Studio 28 或更高版本
- OBS WebSocket 已启用

Windows 若关闭了系统“动画效果”，软件会遵循系统的减少动态效果设置。首次引导会检测并提示，可在 `设置` -> `辅助功能` -> `视觉效果` 中重新开启。
